02 / Ingestion

Як документи потрапляють у Chroma

Ingestion - це підготовка знань для пошуку. Тут сирі файли стають маленькими chunks із metadata, embeddings і стабільними chunk ids.

1. Read files load_text_files(DOCS_DIR) читає тільки .md і .txt.
2. Metadata source, file name, type, length, checksum, modified time.
3. Plan manifest порівнюється з поточними файлами: added, changed, deleted.
4. Split RecursiveCharacterTextSplitter: 300 chars, 60 overlap.
5. Embed OpenAIEmbeddings створює vectors для кожного chunk.
6. Persist Chroma index і ingestion_manifest.json зберігаються локально.

Коли index перевикористовується

Якщо data/index/chroma/ існує, manifest існує, і plan каже needs_rebuild = false, код відкриває готовий Chroma index замість нового rebuild.

Коли index перебудовується

Якщо файл додали, змінили або видалили, система заново ріже всі документи на chunks, перераховує embeddings і записує новий manifest. Це простіше, ніж partial update, і добре для MVP.

03 / Question answering

Шлях одного питання через RAG

І CLI, і API зрештою приходять у answer_question(). Тому важливо розуміти саме service layer у qa_service.py.

1

Validate input

validate_question() прибирає пробіли і відкидає порожнє питання.

2

Normalize chat history

Беруться тільки ролі user/assistant, максимум 6 останніх змістовних повідомлень.

3

Rewrite follow-up question

Якщо є історія, модель переписує питання в standalone form, наприклад “А чим вона відрізняється від ND?” → “Чим MX-5 NA відрізняється від ND?”.

4

Retrieve context

Retriever шукає top-k chunks у Chroma. get_search_k() не просить більше chunks, ніж реально є в collection.

5

Format context and sources

format_context() збирає fragments для prompt, а build_sources() готує список джерел для UI/API.

6

Generate answer

Prompt змушує відповідати українською тільки з retrieved context. Якщо контекст не містить відповіді, очікувана фраза: Я не знайшов цього в документах.

04 / API and frontend

FastAPI тонкий, React показує stream

HTTP layer майже не містить бізнес-логіки. Він приймає запит, викликає service layer і повертає або JSON, або Server-Sent Events для поступового виводу відповіді.

POST /ask

Синхронний endpoint. Повертає повну відповідь і sources одним JSON response.

POST /ask/stream

Streaming endpoint. Спочатку надсилає sources, потім answer_chunk, в кінці done.

React chat

App.jsx читає SSE stream через response.body.getReader() і дописує текст бота chunk за chunk.

Важлива межа: frontend не знає LangChain. Він лише збирає question і chat_history, показує повідомлення та sources. Уся RAG-логіка залишається в backend.

05 / Learning checkpoints

Що варто зрозуміти під час вивчення

D

Document

LangChain Document має текст у page_content і службову інформацію в metadata.

C

Chunk

Малий фрагмент документа. Overlap допомагає не втрачати сенс на межі двох шматків.

V

Vector store

Chroma зберігає embeddings і дозволяє знайти фрагменти, схожі на питання.

P

Prompt boundary

Фінальна відповідь має опиратися тільки на context. Це головний захист від вигаданих фактів.

06 / Tests and eval

Що перевіряється автоматично

Unit tests тут спеціально мережево-незалежні: вони мокають OpenAI і перевіряють маленькі функції. RAG eval окремий, бо він уже залежить від реального індексу, моделі та мережі.

Шар Що покриває Команда
Loader tests читання тільки .md/.txt, metadata, preview make test
RAG helper tests chunk metadata, manifest, ingestion plan, prompts, chat history make test
API tests /ask, /ask/stream, CORS, SSE events make test
RAG eval expected sources, must-include phrases, fallback answer make eval

07 / Practical commands

Команди, які найчастіше потрібні

Backend

make install
make test
make load-docs
make index-demo
make ask QUESTION="Яка різниця між MX-5 NA і ND?"
make api

Frontend

make frontend-install
make frontend-dev
make frontend-build

make test не потребує OpenAI API. А make index-demo, make ask, make eval і make api потребують коректний OPENAI_API_KEY у кореневому .env.

Оновлено як навчальна візуальна карта поточного стану репозиторію. Згенеровані індекси, .env, .venv, node_modules і build artifacts не мають потрапляти в git.
